## Role of Metadata

### Terminological Consistency

The glossary and terminology documents constitute the single source of truth for LAIRM vocabulary. All corpus documents must use the canonical terms defined in these files.

**Example**:
- Use: "autonomous agent"
- Avoid: "agentic AI", "autonomous system", "intelligent agent"

### Source Traceability

The bibliography enables:
- Verification of sources for all factual assertions
- Ensuring academic rigor of the corpus
- Facilitating complementary research

**Citation format**: `[IDENTIFIER-YEAR]`  
**Example**: `[TURING-1950]`, `[BOSTROM-2014]`

### Methodological Transparency

The acknowledgment documents provide:
- Complete transparency about development methodology
- Documentation of human-AI collaboration model
- Demonstration of LAIRM's own governance principles in practice

### Collaborative Governance

The editorial committee ensures:
- Peer review of all documents
- Disciplinary and geographical diversity of contributions
- Quality and consistency of the corpus

## Usage

### For Contributors

1. **Before writing**: Consult the glossary to use correct terminology
2. **During writing**: Cite sources using bibliography identifiers
3. **After writing**: Submit to editorial committee for review
4. **Methodology**: Review acknowledgment documents to understand development process

### For Readers

1. **Unknown terms**: Consult the glossary or terminology documents
2. **Deepen a subject**: Consult bibliographic references
3. **Understand methodology**: Read acknowledgment documents
4. **Contribute**: Consult editorial committee for modalities

## Maintenance

### Document Updates

All metadata documents are living documents, updated as the project evolves:

- **Glossary/Terminology**: Addition of new terms, refinement of definitions
- **Bibliography**: Addition of new references, verification of sources
- **Acknowledgments**: Updates reflecting ongoing development process
- **Index**: Automatic regeneration during compilation

**Update Process**:
1. Proposal for addition/modification via GitHub Issue
2. Review by editorial committee
3. Integration with justification
